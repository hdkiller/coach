This guide outlines the standards for creating both athlete-facing and developer-facing documentation within the Coach Watts platform.

## 1. Directory Structure

All documentation is stored in the `content/documentation/` directory.

- `1.athletes/`: End-user guides (setup, features, metrics).
- `2.developers/`: Technical documentation (API, authentication, architecture).

### Ordering

We use numeric prefixes in filenames (e.g., `1.getting-started.md`) to control the sort order in the sidebar navigation. The prefixes are stripped automatically by the navigation component.

## 2. File Metadata (Frontmatter)

Every Markdown file **must** include a YAML frontmatter block at the top. This data is used for page titles, SEO, and navigation labels.

```yaml
---
title: Page Title (Human Readable)
description: A short summary of what this page covers.
---
```

## 3. Content Guidelines

### Headers and Hierarchy

- **No H1 in Markdown**: Do not use `# My Title` in the content. The layout automatically uses the `title` from your frontmatter as the H1.
- **Start with H2**: Your content should begin with a `## Heading` (H2).
- **Logical Flow**: Use H3 (`###`) for sub-sections. Avoid going deeper than H4 for readability.

### Voice and Tone

- **Athlete Docs**: Empathetic, encouraging, and clear. Avoid overly technical jargon unless explaining a specific metric.
- **Developer Docs**: Technical, precise, and example-driven. Use "Code-First" explanations.

### Code Blocks

Always specify the language for syntax highlighting:

- ` ```bash ` for CLI commands.
- ` ```typescript ` for code snippets.
- ` ```http ` for API request/response examples.

## 4. Components and Layouts

### Tables

Use standard Markdown tables for API parameters or metric definitions.

```markdown
| Parameter | Type | Description            |
| :-------- | :--- | :--------------------- |
| `id`      | UUID | The unique identifier. |
```

### Links

- **Internal Links**: Use relative paths starting from the documentation root, e.g., `[Getting Started](/documentation/athletes/getting-started)`.
- **External Links**: Standard Markdown syntax, e.g., `[Nuxt.js](https://nuxt.com)`.

## 5. New Page Template

Copy and paste this template when creating a new documentation file:

```markdown
---
title: Page Name
description: A concise description of the topic.
---

## Overview

Briefly explain the goal of this guide.

## Section One

Content goes here...

## Section Two

Content goes here...
```
