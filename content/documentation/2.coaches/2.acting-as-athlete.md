---
title: Reviewing an Athlete
description: Understand the current coach-scoped athlete workflow and how it differs from using an athlete account.
---

Coach Watts currently gives coaches dedicated, permission-checked athlete views. You stay in the coach workspace while reviewing an athlete; you do not need their password or account session.

## Open an athlete's coach view

1. Go to **Coaching → Athletes**.
2. Find the athlete and select **Analyze Athlete**.
3. Use the **Overview**, **Calendar**, and **Zones** tabs.

The coach view includes fitness, fatigue, form, readiness, seven-day compliance, recovery signals, upcoming sessions and events, recent completed workouts, and configured power and heart-rate zones.

## What changes are available

Use **Open Coach Calendar** to schedule, move, copy between athlete lanes, inspect, or delete planned sessions through coach-scoped endpoints. Completed workouts open in a read-only coach summary and can be added to workout comparison from the calendar.

Use the library drawers in the coach calendar to schedule workout templates or individual sessions from a plan blueprint.

## About “Act As” mode

The application contains a coaching-context foundation used by some library and API operations, but the current athlete cards and athlete profile do **not** expose a user-facing **Act As** or **Stop Acting As** control. If you do not see that control, use the coach profile and coach calendar workflows described above.

This guide keeps the existing `/acting-as-athlete` address so older links continue to work, but documents the current interface rather than an unavailable button.

## Privacy boundary

Every athlete view verifies an active direct coaching relationship. Team athletes may appear masked to members who do not have staff access or a direct coaching relationship. See [Permissions & Privacy](/documentation/coaches/permissions-privacy).
